## Conventions a new agent must know

- **Docs style guide: `.agents/skills/write-docs/SKILL.md`. Read it fully before editing any page.** Two hard rules bit this session: **no em dashes** (`—`) anywhere in prose, and **Title Case headings**. The repo recently did a deliberate em-dash purge (`6f28064a`, `fb5e65e2`), so reintroducing them is a regression.
- **Ground every claim in source.** Storyteller is a separate repo (`flipbook-labs/storyteller`); read it via `gh api repos/flipbook-labs/storyteller/contents/<path>`. The controls schema lives in `src/controls/` there; the renderer logic in `src/renderers/createManualRenderer.luau` and `src/loadStoryModule.luau`.
- **Docs render code via ` ```code-sample ` path-embeds** pointing at `workspace/code-samples/src/...`. Never hand-write story modules inline; add a real sample and embed it. Edit `src/` only in code-samples; `build/` is generated by `lute run build`.
- **Verify builds:** `cd docs/site && npm run build`, then **grep the log for warnings** (`onBrokenLinks` is `warn`, so a zero exit does not prove links resolve). Run `npm run format` (Prettier) after edits.
- The vault is Obsidian-flavored: wikilinks `[[path|Label]]`, image embeds `![[asset]]`, callouts `> [!note]`. Note transclusion (`![[note]]`) is not supported.
