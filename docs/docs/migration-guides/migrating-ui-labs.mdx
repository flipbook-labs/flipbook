# UI Labs

[UI Labs](https://github.com/ui-labs-rblx/ui-labs) is a storybook plugin for Roblox with its own story format. Flipbook can automatically read most UI Labs stories with no changes required, so you can migrate gradually at your own pace.

## Automatic Migration

When Flipbook loads a story, it detects the UI Labs control format and converts it on the fly. The following control types are migrated automatically:

| **UI Labs**                | **Flipbook Equivalent** | **Notes**                                                                |
| -------------------------- | ----------------------- | ------------------------------------------------------------------------ |
| `UILabs.String()`          | `String`                | Value and default are preserved                                          |
| `UILabs.Number()`          | `Number`                | `Min`/`Max` become `range`, `Step` becomes `step`                        |
| `UILabs.Boolean()`         | `Boolean`               | Value and default are preserved                                          |
| `UILabs.Choose()`          | `Select`                | Items are stringified via `tostring()`; `DefIndex` sets the default      |
| `UILabs.EnumList()`        | `Select`                | Dictionary values become items (sorted); `DefIndex` key sets the default |
| `UILabs.Slider()`          | `Slider`                | `Min`/`Max` become `range`, `Step` becomes `step`                        |
| `UILabs.RGBA()`            | `Color`                 | The `Color` field is extracted; transparency is not preserved            |
| `UILabs.Object()`          | (skipped)               | Instance references are not supported and will be ignored                |
| `UILabs.Datatype.Color3()` | `Color`                 | The `Color3` value is used directly                                      |
| `UILabs.ControlGroup()`    | (flattened)             | Controls inside a group are merged into the top-level table              |
| `UILabs.Ordered()`         | (unwrapped)             | The display order is discarded; the inner control is migrated normally   |

### Choose → Select

`UILabs.Choose` takes `(list, defIndex?)` where `defIndex` is a 1-based array index.

:::warning
During auto-migration, items in a `Choose` list are converted to strings via `tostring()`. If your list contains non-string values (numbers, Color3, etc.), the migrated control values will be strings. When rewriting manually, keep the original types in `items`.
:::

:::note
There is no vanilla equivalent for a dropdown list. Use the Flipbook API to preserve the selection widget.
:::

## Control Groups

`UILabs.ControlGroup` nests controls under a group name. Flipbook does not support nested groups — when auto-migrating, controls inside a group are flattened into the top-level `controls` table. Duplicate keys across groups will be overwritten.

<CodeBlock language="lua" title="ControlGroupUiLabs.luau">
	{ControlGroupUiLabs}
</CodeBlock>

## Unsupported Controls

The following UI Labs control types are silently ignored during auto-migration:

- **`UILabs.Object()`** — Instance references cannot be serialized across stories.
