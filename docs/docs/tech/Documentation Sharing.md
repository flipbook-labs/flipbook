# Documentation Sharing

I'm trying to determine the best place(s) for our documentation and other

writing for our open source project

Types of writing we have:

* Public documentation
    * This goes through the most rigour and is what ultimately appears on our GH
Pages site
* Living documents
    * Things like our Notion workspace. Freeform content that makes no gaurantee
that it's still relevant
* RFCs
    * Things we want to share with the community to get feedback. Slightly more formal than our living documents, but less formal that the public docs

Documentation consumers

1. Me and Pashley
2. Community
    1. Users > Public docs
    2. Powerusers > RFCs, living documents
3. Roblox Internal

Options considered:

4. Notion
    1. Pros: I love Notion so much and it's been so easy to work with, build out
pages and databases
    2. Cons: It's soo expensive. $240/y just for 2 people
5. Google Drive
    3. Pros: Everyone has it
    4. Cons: Harder to organize information
6. GitHub
    5. Pros: Everyone has it. Keeps all of our writing in the same place as our
source code
    6. Cons: Harder to iterate, especially for living docs like some of our
tech/product specs

I think I'm overthinking this. We should be able to use GitHub primarily, which

will naturally bolster our docs in general and build the muscle to edit them.

Code review would not be that burdensome. And for faster iteration you can still

use Notion or any other WYSIWYG editor
